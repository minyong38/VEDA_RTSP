## Q. 왜 Live555 / GStreamer를 안 썼나?

**학습 목적.** RTSP/RTP는 임베디드 영상 분야의 사실상 표준 프로토콜이고,
바이트 단위로 직접 만들어봐야 다음을 이해할 수 있다:

- RTP 12바이트 헤더의 각 비트가 왜 그 위치에 있는지 (M 비트 = 프레임 끝)
- FU-A가 왜 필요한지 (Ethernet MTU 1500 vs H.264 IDR 프레임 수 KB)
- RTSP 상태머신이 왜 그렇게 설계됐는지 (TCP 제어 + UDP 데이터의 분리)

라이브러리는 이걸 다 추상화해버린다.

## Q. RTSP는 TCP, RTP는 UDP — 왜 다른 전송?

- **RTSP (제어)**: 요청-응답이 정확히 도착해야 함 → TCP
- **RTP (영상 데이터)**: 약간의 손실은 허용, 지연이 더 치명적 → UDP
  - TCP의 재전송 대기 > 다음 프레임 도착이면 의미 없음

## Q. C++로 짠 이유는?

- 메모리 레이아웃 직접 제어 (RTP 헤더 비트 패킹)
- POSIX 소켓 API와 매끄럽게 어울림
- 임베디드 타겟에서 런타임 오버헤드 최소
